# EG-PCS Dataset Release Guide

This repository is intended to accompany a public scientific dataset release for
Eye-Tracking-Guided Perceived Cycling Safety (EG-PCS). The dataset contribution is
not only the training code: it includes pairwise perceived-safety labels, the
street-view images used in each comparison, and fixation-based gaze maps derived
from the eye-tracking study.

## Recommended Public Release

Publish a versioned dataset archive on a research data repository that provides a
persistent DOI, such as Zenodo, Dryad, Figshare, or an institutional repository.
The recommended release pattern is:

- Code: GitHub repository.
- Dataset: repository-hosted archive with DOI
  `10.5281/zenodo.20101496`.
- Citation: cite both the paper and the dataset DOI.
- Versioning: use semantic dataset versions such as `v1.0.0`.

Zenodo is the preferred first choice for this project if the final archive fits
within its size limits. If the images make the archive too large, use Dryad,
Figshare+, or an institutional repository with higher storage quotas while
keeping the same directory layout described below.

## Dataset Contents

The public dataset should contain:

- Pairwise comparisons table with labels.
- Street-view image files used by the comparisons.
- Fixation-based gaze maps generated by
  `survey_eye_tracker/build_fixation_based_attention_maps_ogama_like.py`.
- Data dictionary.
- Checksum manifest.
- Dataset license and citation metadata.
- Small loading and validation scripts.

The canonical tabular formats should be CSV and Parquet. The original pickle can
be included for compatibility, but it should not be the only public format.
Pickle is Python-specific and not ideal as a long-term scientific data format.

## Recommended Archive Layout

```text
EG-PCS-Dataset-v1.0.0/
  README.md
  DATASET_CARD.md
  CITATION.cff
  DATA_LICENSE.txt
  data_dictionary.csv
  checksums_sha256.txt

  comparisons/
    comparisons.csv
    comparisons.parquet
    comparisons_df.pickle

  images/
    berlin/
    barcelona/
    london_uk_gov/
    london_uk_collideoscope/
    munich/
    paris/
    sequences/

  gaze_maps/
    864x508/
      *.npy

  scripts/
    load_dataset.py
    validate_dataset_release.py
```

All paths stored in public metadata should be relative to the dataset root. Do
not publish local machine paths such as `/home/csantiago/...`.

## Current Local Dataset Snapshot

The current local `comparisons_df.pickle` contains:

- Rows: 13,623 pairwise comparisons.
- Datasets/cities: `berlin`, `sequences`, `barcelona`, `london_uk_gov`,
  `london_uk_collideoscope`, `paris`, `munich`.
- Labels: `-1` means the left image is preferred, `0` means tie, and `+1`
  means the right image is preferred.
- Source eye-tracking rows: 1,495 comparisons marked with
  `has_eyetracker_source=True`.
- Released valid gaze rows: 1,360 comparisons with both left and right `.npy`
  gaze maps present in the current release folder.
- Gaze map files currently present locally: 2,720 `.npy` files.

These numbers should be regenerated and checked before the final public deposit.

## Core Fields

The comparisons table should include at least:

- `dataset`: city/source subfolder under `images/`.
- `image_l`: left image filename.
- `image_r`: right image filename.
- `score`: pairwise ground-truth label in `{-1, 0, 1}`.
- `score_classification`: optional class label derived from `score`.
- `has_eyetracker`: whether both images in the pair have valid gaze maps.
- `has_eyetracker_source`: original eye-tracking availability flag before
  checking whether both released `.npy` files are present.
- `survey_id`: anonymized survey/session identifier.
- `trial_id`: trial identifier.
- `npy_file_l`: left gaze map filename, if available.
- `npy_file_r`: right gaze map filename, if available.
- `image_l_relpath`: public relative path to the left image.
- `image_r_relpath`: public relative path to the right image.
- `gaze_l_relpath`: public relative path to the left gaze map, if available.
- `gaze_r_relpath`: public relative path to the right gaze map, if available.

## Gaze Map Documentation

The gaze maps should be documented with the following properties:

- File format: NumPy `.npy`.
- Spatial size: currently `864x508`.
- Coordinate convention: aligned with the released image pixel coordinate system.
- Generation script:
  `survey_eye_tracker/build_fixation_based_attention_maps_ogama_like.py`.
- Meaning: fixation-based attention/heatmap values aggregated from the
  eye-tracking experiment.
- Normalization: document the exact final convention used by the generation
  script before release, for example sum-normalized density, max-normalized
  heatmap, or raw fixation accumulation.

## Ethics, Privacy, and Rights

Before publishing the final dataset, confirm:

- Human-subject consent allows public release of derived gaze maps.
- Raw personally identifying eye-tracking records are not included unless
  explicitly approved.
- Survey IDs are anonymized or hashed.
- Image redistribution rights are clear.

If image redistribution rights are not available, publish the labels, gaze maps,
image identifiers/URLs, and a reconstruction script instead of republishing the
images directly.

## Licensing Recommendation

Use a permissive scientific data license when possible:

- Pairwise labels and gaze maps: `CC BY 4.0` is recommended.
- Code: use a code license such as MIT, BSD-3-Clause, or Apache-2.0.
- Images: use the license allowed by the original image source. Do not apply a
  license that conflicts with the image provider's terms.

The final dataset archive should contain an explicit `DATA_LICENSE.txt` that
states which license applies to each component.

## Creating a Release Package

Use the release preparation script:

```bash
python scripts/dataset/prepare_dataset_release.py \
  --comparisons comparisons_df.pickle \
  --images-root images/printart/subjectivesafety_images \
  --gaze-root survey_eye_tracker/Eyetracker_attention_maps/864x508 \
  --output-dir .dataset_releases/EG-PCS-Dataset-v1.0.0 \
  --copy-assets \
  --hardlink-assets \
  --write-checksums
```

Then validate it:

```bash
python scripts/dataset/validate_release.py .dataset_releases/EG-PCS-Dataset-v1.0.0
```

After validation, compress the folder and upload it to the chosen data
repository. Record the DOI in both the repository README and the dataset
`CITATION.cff`.

For a Zenodo API upload, first create a Zenodo access token with deposit
permissions and export it as `ZENODO_TOKEN`. Then run:

```bash
python scripts/dataset/upload_to_zenodo.py \
  .dataset_releases/EG-PCS-Dataset-v1.0.0.tar.gz \
  --metadata docs/dataset/zenodo_metadata.json
```

The script creates a draft upload and stops before publishing. Review the draft
on Zenodo, copy the reserved DOI into `CITATION.cff`, regenerate the release
archive, and publish only when the files and metadata are final.

The public dataset release intentionally does not include the experimental
train/validation/test split files. Those can remain in the code repository or be
regenerated by researchers from the comparison table when needed.
